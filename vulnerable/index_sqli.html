🔍 什麼是 SQL Injection？

📖 定義

這是一種發生在應用程式資料層的安全漏洞。當程式未對用戶輸入的資料進行適當過濾或轉義，並將其直接拼接至 SQL 查詢語句中時，攻擊者可藉此改變原有的查詢邏輯。

⚠️ 核心問題

程式無法區分「開發者預期的指令」與「用戶輸入的惡意資料」，導致資料庫執行了未經授權的指令。

💥 影響範圍

攻擊者可藉此繞過身份驗證、竊取敏感資料（如密碼、個資）、篡改或刪除數據，甚至在某些配置下取得伺服器的作業系統控制權。

🎭 攻擊情境：後台登入系統

假設有一個簡單的後台登入系統，透過 username 和 password 驗證用戶。

✅ 正常請求
admin
password123
生成的 SQL 語句
SELECT * FROM users WHERE username = 'admin' AND password = 'password123';
結果：資料庫回傳該用戶資料，登入成功。
😈 攻擊輸入
' OR '1'='1' #
隨意輸入
邏輯篡改後的 SQL
SELECT * FROM users WHERE username = '' OR '1'='1' #' AND password = '...';

觸發攻擊：由於 # 在 MySQL 中代表註解，因此後面 AND password = ... 的密碼驗證部分被無效化了！

條件 '1'='1' 恆為真 (True)，伺服器將回傳資料表中第一個用戶（通常是管理員），攻擊者無需密碼即可登入！

⚖️ 程式碼大對決：危險寫法 vs 安全寫法

login.php (Vulnerable Code - PHP) ❌

使用 mysqli 直接拼接字串的典型錯誤：


$conn = mysqli_connect("localhost", "db_user", "db_pass", "database");
$username = $_POST['username'];
$password = $_POST['password'];

// 🚨 危險：直接將變數拼接到 SQL 字串中
$query = "SELECT * FROM users WHERE username = '$username' AND password = '$password'";

$result = mysqli_query($conn, $query);

if (mysqli_num_rows($result) > 0) {
    echo "登入成功！";
} else {
    echo "帳號或密碼錯誤。";
}
secure_login.php (Secure Code - PHP) ✅

使用 預編譯陳述式 (Prepared Statements) 與 PDO：


try {
    $pdo = new PDO("mysql:host=localhost;dbname=database", "db_user", "db_pass");
    
    // ✨ 步驟 A：編寫具備佔位符 (?) 的 SQL 模板
    $sql = "SELECT id, username FROM users WHERE username = ? AND password = ?";
    $stmt = $pdo->prepare($sql);
    
    // ✨ 步驟 B：綁定參數並執行（資料與指令分離）
    $stmt->execute([$_POST['username'], $_POST['password']]);
    $user = $stmt->fetch();
    
    if ($user) {
        echo "登入成功，歡迎 " . htmlspecialchars($user['username']);
    } else {
        echo "認證失敗。";
    }
} catch (PDOException $e) {
    die("資料庫錯誤。");
}

🛡️ 防禦之道：開發實戰建議

從資訊管理與系統架構的角度，防禦 SQL Injection 應遵循「深度防禦」原則：

🔒

全面參數化

所有的資料庫互動（SELECT, INSERT, UPDATE, DELETE）皆必須使用預編譯陳述式。這是目前公認最有效的防線。

✅

輸入驗證 (白名單)

採取「白名單」策略。例如，若預期輸入是數字（如 user_id），應在進入資料庫前強制轉換型別 (int)$_POST['id']。

🔑

最小權限原則

網頁應用程式連接資料庫的帳號，不應具備 DROP TABLE 或存取系統表（如 information_schema）的權限。應針對不同模組分配僅具備所需權限的帳號。

🔇

避免詳細錯誤訊息

在正式環境中，應關閉資料庫錯誤顯示（如 display_errors = Off），避免攻擊者透過錯誤回顯（Error-based SQLi）推測資料庫結構。

🛠️

使用 ORM 工具

導入如 Eloquent (Laravel) 或 Doctrine 等 ORM 框架，這些工具底層預設即使用參數化查詢，能大幅降低人為疏忽導致的漏洞。
