進階篇：時間盲注

⏱️ 什麼是 Time-Based (時間盲注)？

👻 隱蔽的攻擊

這是一種更隱蔽、更耗時的 盲注 (Blind SQL Injection) 技術。當前端不顯示錯誤訊息，也不因查詢結果改變畫面時，駭客就會使用此招。

⏳ 核心原理

利用 SQL 的時間延遲函數 (如 SLEEP())，根據網頁「回應速度」的快慢，來推斷資料庫中的隱藏資訊。

🎯 攻擊範例

假設駭客想知道資料庫名稱的第一個字是否為 'a'：
AND (SELECT IF(SUBSTRING(database(),1,1)='a', SLEEP(5), 1))
若等了 5 秒才回應，代表猜對了！若瞬間回應則代表猜錯。

⚖️ 程式碼大對決：Time-Based 防禦

time_login.php (容易受延遲攻擊 ❌) 

$id = $_POST['id'];

// 🚨 危險：直接將變數拼接到 SQL 字串中
$sql = "SELECT name FROM products WHERE id = $id";

$result = $conn->query($sql);
time_secure.php (使用 PDO 預處理 ✅) 

$id = $_POST['id'];

// ✨ 安全：使用預編譯陳述式，SLEEP() 會被當作純字串
$stmt = $pdo->prepare("SELECT name FROM products WHERE id = ?");
$stmt->execute([$id]);

🛡️ 防禦之道：防止時間盲注

⏱️

設定請求超時 (Timeout)

在網頁伺服器或 PHP 層級設定合理的 Timeout（例如 30 秒），防止資料庫連線池被駭客以長延遲攻擊掛起，導致正常服務癱瘓。

📈

監控慢查詢 (Slow Logs)

定期檢查資料庫的 Slow Query Logs。如果發現大量帶有 SLEEP() 或異常延遲的 SQL 記錄，通常就是正在遭受盲注攻擊的徵兆。
