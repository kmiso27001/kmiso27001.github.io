實戰篇：環境特徵外洩

🖥️ 什麼是 Server 標頭外洩？

🔍 技術清單大公開

Web 伺服器 (如 Apache 或 Nginx) 預設會在 HTTP 響應的 Server 標頭中，附上自身軟體名稱、詳細版本號、甚至作業系統和外掛模組的資訊。

⚠️ 一鍵攻擊腳本

駭客只需發送一個請求，就能得知伺服器使用的精確版本 (如 Apache 2.2.15)。接著就能在 Exploit-DB 上尋找該版本專屬的「一鍵攻擊腳本 (One-Click Exploits)」直接打擊。

🛡️ 最小資訊揭露

基於資安原則，系統不應回傳任何對正常用戶無益、但對攻擊者有利的技術細節。隱藏版本號雖然不能修補漏洞，但能大幅增加駭客掃描與測試的成本。

🕹️ 實戰演練：自動化掃描器模擬

下方模擬駭客使用掃描工具對伺服器發送請求。請點擊「😈 啟動掃描器」來觀察伺服器回傳的 Server 標頭，看看預設配置與安全配置的差異！

❌ 測試危險配置 (預設暴露所有細節)
> curl -I https://shop.com/api
等待掃描...
✅ 測試安全配置 (僅顯示產品名稱)
> curl -I https://shop.com/api
等待掃描...

⚖️ 伺服器配置大對決：如何收斂資訊？

Apache / Nginx (危險的預設配置 ❌) 

# Apache (httpd.conf)
# 預設通常為 Full 或 OS，會顯示版本、OS 與模組資訊
ServerTokens Full
ServerSignature On

# Nginx (nginx.conf)
# 預設為開啟，會在 Header 與錯誤頁面顯示精確版本號
server_tokens on;
Apache / Nginx (安全的防護方式 ✅) 

# ✨ Apache (httpd.conf)
# 僅顯示軟體名稱 "Apache"，隱藏版本號與作業系統
ServerTokens Prod
ServerSignature Off

# ✨ Nginx (nginx.conf)
# 隱藏 Nginx 的具體版本號
server_tokens off;

🛡️ 維運與治理建議：消除系統指紋

📉

遵循最小資訊揭露原則

伺服器不應回傳任何對正常用戶無益的技術細節。將 ServerTokens 設為 Prod 是伺服器上線前資安稽核中最基本的要求。

🎭

進階偽裝 (WAF / Reverse Proxy)

在進階架構中，可以使用負載平衡器或反向代理 (如 Nginx) 將 Server 標頭重寫為不具名內容 (例如 Server: WebServer)，徹底消除特徵。

🧹

錯誤頁面清理

除了 Header 之外，還要確保 404 或 500 錯誤頁面是由應用程式自定義的，避免系統預設錯誤頁面的底部洩漏伺服器的技術簽名。

🤖

自動化配置檢核 (IaC)

將伺服器配置納入「基礎架構即代碼 (Infrastructure as Code)」的審核流程中，使用自動化指令碼檢查 server_tokens 是否已確實關閉。
