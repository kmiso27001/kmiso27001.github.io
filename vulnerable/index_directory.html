漏洞探測：Directory Browsing

📁 什麼是目錄遍歷 (Directory Browsing)？

⚙️ 伺服器配置失誤

當使用者請求一個沒有預設首頁（如 index.php）的目錄時，若伺服器開啟了自動索引 (Auto-indexing)，就會將該目錄下的所有檔案列出來。

🕵️‍♂️ 駭客的尋寶遊戲

駭客常會探測 /config, /backup, /uploads 等目錄。若這些目錄開啟了目錄遍歷，駭客就能一覽無遺，尋找有價值的目標。

💥 致命的資訊洩漏

工程師常因為貪圖方便，把資料庫設定檔 db.bak 或 secret.txt 備份在網頁目錄下。一旦目錄被看光，整個系統等於雙手奉上！

⚖️ 伺服器配置大對決：目錄遍歷防禦

危險配置 (Apache .htaccess / httpd.conf) 

# 🚨 危險：'+Indexes' 啟用了自動目錄列表功能
Options +Indexes +FollowSymLinks

# Nginx 的寫法 (nginx.conf):
# location / {
#     autoindex on;
# }
安全配置 (Apache .htaccess / PHP) 

# ✨ 方法 A：在 Apache 伺服器層級明確禁用索引 (推薦)
Options -Indexes

# Nginx 的寫法:
# autoindex off;


<!-- ✨ 方法 B：在不希望被遍歷的目錄放一個 index.php -->
<?php
// 強制回傳 403 並停止執行
header("HTTP/1.1 403 Forbidden");
exit("Access Denied");
?>

🛡️ 開發建議：如何避免檔案結構洩漏

⛔

最小化暴露面

確保生產環境（Production）中，所有 Web 伺服器的 Auto-indexing 功能皆已從設定檔（如 apache2.conf 或 nginx.conf）全域關閉。

📦

敏感檔案路徑隱蔽

設定檔（如 config.php）、備份檔（如 .bak）不應存放在 Web Root（網頁對外公開的根目錄）下，應移至上層目錄，讓瀏覽器絕對無法直接存取。

🔐

PHP 動態權限檢查

如果目錄存放的是使用者上傳的機密檔案，不應直接開放 URL。應透過 PHP 腳本驗證使用者 Session，再使用 readfile() 輸出檔案內容給前端。

📡

自定義錯誤頁面與掃描

配置統一的 403 / 404 錯誤頁面，隱藏伺服器版本資訊。並定期使用 OWASP ZAP 或 Nikto 等工具進行自動探測掃描。
